## 1. Fable 5 時代の再定義（12タスク一括改修）

- **judge をモデル非依存化**：「Opus 4.8」ハードコードを全廃し「judge=セッション最強モデル、パネル=安価 Claude（model=opus 明示指定）＋異種ベンダー」の構図に。panel.md に「judge は最強モデルに座らせる」節を追加。README の物語も「Fable 級を合成する」→「単一ベンダーで消せない系統的盲点への保険」に書き換え。
- **judge の質**：匿名ラベル分析（回答A/B/…→監査証跡で実名復元）、事前コミット（precommit.md→差分を監査証跡に常設）、接地対称性の注記。
- **インフラ**：runs/ ディレクトリへの生入出力保存、validate_json.sh（JSON 出力の決定論ゲート）、tests/（純 bash・API 不要）。
- **修正**：quorum-shell 負インデックスバグ、プロンプト/APIキーの argv 露出（codex/gemini stdin 化・curl config 化。grok CLI のみ argv 不可避で注記）。

## 2. 実走検証と改善ループ

- **/quorum-opus 初実走**（opus×2、LLM-as-judge バイアスの調査問題）→ パイプライン全機構の動作確認。発見3件を即修正：引用 spot-check 手順（rubric が出典を無検証で加点していた）、mapping.txt を実名復元の正とする、パネリスト回答内の指示文に従わない（インジェクション耐性）。
- **フルパネル実験**（grok＋opus×3、自己選好誘発問題）で「匿名化の実効性」「文体正規化の要否」を検証：
  - 盲判定 vs 実名判定：総合順位完全一致・スコア差±1 → ラベルは判定を歪めなかった（n=1）
  - 文体指紋：正規化前は不正解（文体のみ確信度≒偶然）。正規化後の正解は「同族 N-1 の内容収束による消去法」＋正規化アーティファクトによるもの
  - **結論：文体正規化は不採用**（SKILL 明記）。匿名化はラベル遮断の規律として維持
  - **副発見**：grok CLI が実走中に呼び出し元リポを読んでいた → run スクリプトの CLI 経路を空一時ディレクトリで実行するよう修正（実機確認済み）
