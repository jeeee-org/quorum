# 2026-07-10 checkpoint

## codex パネリストを GPT-5.6 Sol に明示固定

### 経緯
- ユーザーが「GPT-5.6 はまだ出ていないと言われたが実際は出ている」（Yahoo/ZDNET Japan 記事、07-10 配信）と指摘。
- 調査の結果、GPT-5.6 は **2026-07-09（米国時間）一般公開**（3層: Sol/Terra/Luna）。当方の知識カットオフ 2026-01＋公開直後で web 未反映気味だったのが「見つけられなかった」理由。断定は誤りで「この環境からは見えない」と言うべきだった。
- ユーザー環境で `/model` に 5.6 が出ない件を切り分け:
  1. 貼られた picker は旧 `v0.130.0` セッションの表示（5.6 表示には 0.144+ が必要）。
  2. その後 `npm install -g @openai/codex` 済み → **ディスク上は 0.144.1**。新規起動で 5.6 が picker に出た。
  3. `~/.codex/models_cache.json` も 23:44 に更新され、正式 ID を取得: `gpt-5.6-sol` / `gpt-5.6-terra` / `gpt-5.6-luna`。

### 発見（重要）
- `run_codex.sh` は `-m` 未指定で **codex 既定モデルに丸乗り**していた。
- 5.5 で動いていたのは各PCの `~/.codex/config.toml` の `model = "gpt-5.5"` pin 頼み。**スクリプト単体では素性を保証していなかった**。
- CLI 0.130 の picker で確認すると **素の既定は `gpt-5.3-codex`（コーディング特化）**。config pin の無い他PCでは 5.5 でも 5.6 でもなく 5.3-codex に化ける潜在バグ。

### 対応
- `run_codex.sh` の `codex exec` に `-m gpt-5.6-sol` を追加（config 依存を排し配布物として自己完結）。
- 表記を全面更新: `run_codex.sh` ヘッダー / SKILL.md（description・パネル表）/ README.md（3箇所）。旧「GPT-5.5・0.130.0」→「GPT-5.6 Sol・0.144.1」。
- モデル選択はユーザー確認の上 **Sol（最上位・最大コスト）** を採用。

### 検証（codex-cli 0.144.1）
- `QUORUM_ENABLE_CODEX=1 run_codex.sh --check` → exit 0。
- 実プロンプト実行 → セッションログに `"model":"gpt-5.6-sol"` 記録を確認。
- negative test: `-m gpt-5.6-nonexistent` は exit 1 で失敗 → `-m` が honored であることを確認。
- 注意: モデルの自己申告（「GPT-5 Codex」等）は不正確。素性確認はセッションログ／negative test で行う。
