# Judge ルブリック（突き合わせ → 統合）

メイン Opus 4.8 が全パネリストの回答を以下の軸で構造化する。**平均や多数決ではない**——食い違いと盲点を能動的に洗い出すのが目的。

## 分析軸

| 軸 | 見るもの |
|---|---|
| **Consensus** | 複数パネリストが一致している点。確度が高い土台 |
| **Contradictions** | 食い違い。**どちらがより確からしいか**を根拠（出典・論理）つきで判定する。安易に折衷しない |
| **Partial coverage** | 一部のパネリストしか触れていない論点。抜けの補完 |
| **Unique insights** | 単独パネリストだけが出した有用な洞察。捨てずに拾う |
| **Blind spots** | 全員が見落としていそうな点。judge が能動的に補う |

- 各項目は**どのパネリスト由来か**を明示する（追跡可能性）。
- 矛盾の解決を最優先する。「両論併記で逃げる」のは最後の手段。

## 最終出力の形

1. **最終回答**（本体・トップ）
   - 上の分析を根拠に、judge が一貫した1つの答えを書く。
2. **監査証跡（audit trail）**（畳んで添付）
   - Consensus / Contradictions / Partial / Unique / Blind spots を、パネリスト帰属つきで簡潔に。
   - 読み手が「なぜこの結論か」を後から検証できる粒度にする。

## judge の心構え

- パネリストの一致＝正しさ、ではない。**全員が同じ間違いをすることもある**（共通バイアス）。Blind spots で能動的に疑う。
- 出典・再現性・論理の強さで重みづけする。多数決に流されない。
