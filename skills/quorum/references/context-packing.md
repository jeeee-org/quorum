# context-packing — quorum に渡す前の「司書」手順

quorum 自体は材料を自動収集しない。外部CLIパネリストに届くのは `$PROMPT` のテキストだけで、
呼び出し元のリポやライブ状態は見えない。ネイティブ・サブエージェントにも同じ材料だけを渡して
比較条件を揃える。∴ **「何を入れて何を捨てるか」はメインセッション（＝司書）の責任**。
巨大MD・複数ファイル・MCP のライブ状態に判断材料が散らばっている時、
fan-out（SKILL.md step 2）の前に `$PROMPT` を**自己完結**に組み上げる手順を定型化する。

## いつ使うか
- **使う**: 高ステークスで、判断材料が巨大MD／複数ファイル／ライブ状態に散在（SRE 型）。
- **使わない**: 低ステークス、または材料が小さく問い単体で完結する時は、素の問いをそのまま渡す。

## 司書の5手順
1. **決定文を1〜2文に固める** — 「何を判断したいか」「成功条件・制約は何か」。ここがブレると全員が別の問いに答える。
2. **広く集める** — `grep`/`glob`/`read` ＋ インフラ閲覧スキル ＋ MCP で関連箇所を**広めに**取得。
3. **絞る（最重要）** — トークン予算を決め、**決定を動かす断片だけ**残す。各断片に出典（`file:line` or コマンド名）を付ける。
4. **context pack を組む** — 下のテンプレで自己完結化。パネリストは前提も背景も知らない前提で書く。
5. **渡す** — `$PROMPT = 決定文 + context pack`。`/quorum` に流す（fan-out 以降は SKILL.md 通り）。

## 絞り込みの原則
- **関連性 > 網羅性**。決定を動かさない情報は載せない（コスト N 倍＋長文脈劣化＝context rot）。
- **出典必須**。各事実に `file:line` かコマンド名。パネリストが「裏取り済みの事実」と「司書の仮説」を区別できる。
- **鮮度を明示**。ライブ状態（MCP 由来など）は**取得時刻**を添える（例 `@2026-06-16T10:00`）。
- **矛盾はそのまま渡す**。司書が解決せず「ここが食い違う」と明示 → judge の Contradictions / Blind spots が働く（解決済みにすると盲点を潰せる利点が消える）。
- **機密は出さない**。外部CLIに渡る前提（学習・キャッシュされ得る）。シークレット・PII・社外秘はマスク or 除外。← SRE 環境では特に必須。
- **落としたものを1行残す**。「大量にあるログのうち直近1hのみ採用」等。無音の切り捨てはしない（SKILL の `panel.dropped` と同じ精神）。

## context pack テンプレ（コピペ）
```
## 判断したいこと（決定文）
<1〜2文。成功条件・制約も書く>

## 前提・背景（パネリストは何も知らない前提で）
<対象システムの一言説明／効いてくる制約>

## 証拠（出典つき・絞り込み済み）
- [src: path/to/runbook.md:120-145] <要点だけ抜粋>
- [src: $ kubectl get deploy -n prod @2026-06-16T10:00] <ライブ状態の要点>
- ...

## 既知の食い違い・不明点
- <MD 同士の矛盾／未確認事項／取得できなかったもの>

## 問い
<答えてほしいこと。判断・リスク・盲点を求める形にする>
```

## SRE 環境での当て方（insights #10「司書＋専門家会議」）
- **司書＝メインホスト（Claude Code / Codex）**：利用可能なツールでライブ状態、巨大 MD を retrieval。上の5手順で pack 化。
- **quorum＝専門家会議**：pack を全パネリストに同一プロンプトで配り、judge が Consensus/Contradictions/Blind spots を出す。
- **将来 TAKT へ移す時**：pack 化（手順2-4）は TAKT の step 入力整形に**そのまま移植**できる。retrieval を MCP に寄せておくほど移植が楽（戦略投資は MCP）。

## 限界
- パネリストは **pack の外を見られない**。pack の質＝回答の上限。重要な証拠の取りこぼしは judge でも回復しない。
- **大きすぎる pack は逆効果**（context rot）。1パネリストの実効コンテキスト窓に収める。入りきらないなら「絞りが甘い」サイン。
- 司書（メイン）が retrieval で握った事実誤りは全パネリストに伝播する。出典つきにして**パネリストが疑える**余地を残す。
